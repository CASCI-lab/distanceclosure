Dijkstra algorithm¶

Implementation of a generalized version of the Dijkstra algorithm using Heap Queue and multiprocessing to compute transitive closure. This algorithm uses a modification to the path length to compute both metric and ultrametric closure.

Warning

There is no need for this class to be called directly, since it will be called from closure().

Note

A very good tutorial of how the Djikstra algorithm works can be seen here.

all_pairs_shortest_paths(kind='metric', n_jobs=1)¶

Computes All Pairs Shortest Paths (APSP).

Parameters:
  • kind (string) – The metric type. ‘metric’ or ‘ultrametric’ are currently accepted.
  • n_jobs (int) – The number of CPUs to use to do the computation. -1 means ‘all CPUs’.
Returns:

shortest_distances – A dict-of-dicts of distances between all pair of nodes. Ex: {'a':{'c':0.1}} shortest_complete_paths (dict): A dict-of-dicts-of-list of the shortest path between all pair of nodes. Ex: {'a':{'c':['a','b','c']}}
Return type:

dict

Note

The same as calling single_source_shortest_paths() for every node as a source.
